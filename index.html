1. Дан код:

d = b++; alert(d); // 1 (ответ: сначала переменной d присваивается значение b = 1, затем b увеличивается на 1 инкрементом в постфиксной форме)
c = (2+ ++a); alert(c); // 5 (ответ: a уже = 2, первым действием она увеличивается еще на 1 инкрементом в префиксной форме, теперь а = 3; вторым действием складывается с 2 и результат = 5 присваивается переменной c)
d = (2+ b++); alert(d); // 4 (ответ: b = 2 складывается с 2 = 4 и результат присваивается d, на выходе b++ инкрементно увеличивается до 3, что мы и видим в последней строке)
alert(a); // 3 (ответ: к переменной а, изначально равной 1, дважды применялось увеличение на единицу на входе в арифетические выражения, результат = 3)
alert(b); // 3 (ответ: к переменной b, изначально равной 1, дважды применялось увеличение на единицу после выхода из расчета арифетического выражения, в итоге результат = 3)

### 7. *Сравнить null и 0. Попробуйте объяснить результат.
ответ: в строках 124-131 файла js2.js + Статья на хабре

null будет преобразовано в +0, а 0 останется самим собой. Ни одно из этих значений не является NaN. Значение +0 равно 0, в результате алгоритм возвращает false. Таким образом:
null > 0; // false
null < 0; // тоже false
Пытаясь понять, равно ли значение null значению 0, мы сразу переходим из шага 1 к шагу 14, так как Тип(x) отличается от Типа(y). Как ни странно, но шаги 14-21 тоже к нашему случаю не подходят, так как Тип(х) — это null. Наконец мы попадаем на шаг 22, после чего false возвращается как значение по умолчанию!
В результате и оказывается, что:
null == 0; //false
Если null < 0 принимает значение false, то null >= 0 принимает значение true.
В результате мы и получаем:
null >= 0; // true
И, на самом деле, в этом есть смысл. С точки зрения математики, если у нас есть два числа, x и y, и если x не меньше, чем y, тогда x должно быть больше чем y или равно ему.
